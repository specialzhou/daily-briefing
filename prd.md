## 1. 产品概述

### 1.1 产品定位
面向跨境支付行业技术从业者的**专业成长型每日简报系统**，聚合加密货币投资、AI 工具情报、跨境支付专业知识三大核心板块，帮助用户高效获取高质量信息并转化为行动。

### 1.2 目标用户
- **主要用户**：跨境支付公司技术 TL/工程师（如 PayerMax 等技术团队）
- **用户特征**：
  - 需要持续关注加密货币市场动态
  - 希望高效利用 AI 工具提升工作效率
  - 系统性学习跨境支付业务知识
  - 时间宝贵，需要精简但有深度的内容

### 1.3 核心价值
| 价值维度 | 描述 |
|----------|------|
| 信息聚合 | 一站式获取多领域高质量信息，避免信息碎片化 |
| 行动导向 | 每条信息附带"行动建议"，从知道到做到 |
| 系统学习 | 跨境支付训练营按知识体系循序渐进，培养专业能力 |
| 时间效率 | 精简摘要 + 原文链接，快速浏览或深度阅读可选 |

## 2. 功能需求

### 2.1 内容板块

#### 2.1.1 加密货币投资分析
| 字段 | 描述 |
|------|------|
| 内容 | 行情分析、投资机会、投资建议及理由 |
| 数据源 | 币安交易所 (Binance) API |
| 更新频率 | 每日 |
| 展示要素 | 价格趋势、关键指标、投资建议、行动建议、信息来源链接 |

#### 2.1.2 OpenClaw 生态推荐
| 字段 | 描述 |
|------|------|
| 内容 | 高赞用法、插件、Skill 推荐及理由 |
| 数据源 | GitHub、官方文档、社区讨论 |
| 更新频率 | 每日 |
| 展示要素 | 推荐内容、推荐理由、行动建议、来源链接 |

#### 2.1.3 Claude Code 高赞用法
| 字段 | 描述 |
|------|------|
| 内容 | 高赞用法推荐及理由 |
| 数据源 | 官方信息源、社区高热讨论 |
| 更新频率 | 每日 |
| 展示要素 | 用法说明、推荐理由、行动建议、来源链接 |

#### 2.1.4 每日 AI 情报
| 字段 | 描述 |
|------|------|
| 内容 | AI 领域最新动态、工具发布、技术突破 |
| 数据源 | 官方信息源 + Twitter/Reddit/Discord 高热讨论 |
| 更新频率 | 每日 |
| 展示要素 | 情报摘要、对用户价值分析、行动建议、来源链接 |

#### 2.1.5 跨境支付训练营 ⭐ 核心板块
| 字段 | 描述 |
|------|------|
| 内容 | 系统性跨境支付业务知识课程 |
| 知识体系 | 支付渠道、支付能力产品包装、商户入驻、交易支付流程、异常交易处理、资金清结算、对账、外汇流动性等 |
| 数据源 | 专业领域博主（如陈天宇宙）、行业报告、官方文档 |
| 更新频率 | 每日 |
| 展示要素 | 课程主题、知识内容、案例分析、行动建议、来源链接 |
| 特色 | 按知识体系循序渐进，形成完整学习路径 |

### 2.2 推送与访问

| 需求项 | 规格 |
|--------|------|
| 推送时间 | 每天早上 8:00 (UTC+8) |
| 推送频率 | 每日，不区分工作日/周末 |
| 访问方式 | 公网网页链接 |
| 部署平台 | Vercel |
| 关联仓库 | GitHub: daily-briefing |

---

### 2.3 交互功能

#### 2.3.1 标记已读
- 用户可标记单条内容或整日简报为"已读"
- 已读内容视觉区分（如灰色、删除线）
- 支持批量标记

#### 2.3.2 收藏功能
- 用户可收藏感兴趣的内容
- 收藏内容独立页面查看
- 支持取消收藏

#### 2.3.3 笔记功能
- 用户可在任意内容下添加个人笔记
- 笔记支持 Markdown 格式
- 笔记与内容关联，可随时查看

#### 2.3.4 任务转化
- 用户可将内容转化为待办任务
- 任务可设置优先级、截止日期
- 任务状态追踪（待办/进行中/已完成）

#### 2.3.5 历史归档
- 所有历史简报自动归档
- 支持按日期浏览历史简报
- 支持关键词搜索历史内容
- 支持按板块筛选历史内容

## 3. 技术需求

### 3.1 前端技术栈
| 技术 | 推荐方案 |
|------|----------|
| 框架 | Next.js (适配 Vercel 部署) |
| UI 库 | Tailwind CSS + shadcn/ui |
| 状态管理 | React Context / Zustand |
| 数据存储 | LocalStorage (v1.0) → Supabase/PostgreSQL (v1.1+) |

### 3.2 后端/数据获取
| 功能 | 实现方案 |
|------|----------|
| 数据抓取 | Next.js API Routes / Serverless Functions |
| 定时任务 | Vercel Cron / GitHub Actions |
| 数据存储 | 每日简报内容存储为 Markdown/JSON 文件 |

### 3.3 数据源集成
| 数据源 | 接入方式 |
|--------|----------|
| 币安 API | 公开 API (https://api.binance.com) |
| Twitter | Twitter API v2 / 第三方服务 |
| Reddit | Reddit API |
| Discord | Discord API / 爬虫 |
| GitHub | GitHub API (使用提供的 Token) |

## 4. 验收标准

### 4.1 功能验收
| ID | 功能 | 验收标准 |
|----|------|----------|
| F1 | 每日简报生成 | 每天早上 8 点自动生成当日简报 |
| F2 | 五大内容板块 | 加密货币、OpenClaw、Claude Code、AI 情报、跨境支付均正常展示 |
| F3 | 信息源链接 | 每条内容附带可点击的来源链接 |
| F4 | 标记已读 | 可标记内容已读，状态持久化 |
| F5 | 收藏功能 | 可收藏/取消收藏，收藏列表可查看 |
| F6 | 笔记功能 | 可添加/编辑/删除笔记 |
| F7 | 任务转化 | 可将内容转为任务，任务状态可追踪 |
| F8 | 历史归档 | 可查看历史简报，支持搜索和筛选 |
| F9 | Vercel 部署 | 可通过公网链接访问 |

### 4.2 性能验收
| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 3 秒 |
| 数据更新延迟 | 简报生成后 5 分钟内可访问 |
| 移动端适配 | 支持主流移动浏览器 |

### 4.3 安全验收
| 项目 | 要求 |
|------|------|
| GitHub Token | 不暴露在前端代码中 |
| API 密钥 | 存储在环境变量 |
| 用户数据 | LocalStorage 加密存储（如敏感） |
